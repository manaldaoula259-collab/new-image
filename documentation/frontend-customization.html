1. Logo and Images

Update your branding assets:

Logo

  • Header Logo: Update in src/components/layout/Header.tsx
  • File Location: Replace logo image in public/ directory
  • Code: 
    <Image
  src="/logo.svg"  // Update path to your logo
  alt="MyDzine"
  width={120}
  height={40}
/>

Favicon and Icons

  • Replace public/favicon.ico with your favicon
  • Update public/apple-touch-icon.png for iOS devices
  • Update public/android-chrome-*.png for Android

Tool Preview Images

  • Update tool preview images in public/tool-*.png
  • Each tool has a preview image (e.g., tool-ai-anime-generator.png)
  • Images should be 800x600px for best quality

ğŸ¨ Color Scheme & Theme

Chakra UI Theme Configuration

The theme is configured in src/styles/theme.ts. Customize colors, fonts, and component styles:

import { extendTheme } from "@chakra-ui/react";

const theme = extendTheme({
  config: {
    initialColorMode: "dark",
    useSystemColorMode: false,
  },
  colors: {
    brand: {
      50: "#f8fafc",
      100: "#f1f5f9",
      // ... customize brand colors
      500: "#1a1a1a",  // Primary brand color
      600: "#0a0a0a",
      700: "#000000",
    },
  },
  semanticTokens: {
    colors: {
      "bg.canvas": { default: "#05060f" },
      "bg.surface": { default: "rgba(11, 17, 28, 0.92)" },
      "text.primary": { default: "#e2e8f0" },
      "text.muted": { default: "#94a3b8" },
      // ... customize semantic tokens
    },
  },
  // Component styles
  components: {
    Button: {
      variants: {
        brand: {
          bg: "var(--accent)",
          color: "white",
          // ... custom button styles
        },
      },
    },
  },
});

export default theme;

CSS Variables

The theme uses CSS variables for consistent styling across components:

:root {
  --bg-canvas: var(--chakra-colors-bg-canvas);
  --bg-surface: var(--chakra-colors-bg-surface);
  --text-primary: var(--chakra-colors-text-primary);
  --text-muted: var(--chakra-colors-text-muted);
  --accent: var(--chakra-colors-brand-500);
  --border-subtle: var(--chakra-colors-border-subtle);
  /* ... more variables */
}

Customize Colors

  1. Update brand colors in src/styles/theme.ts
  2. Modify semantic tokens for backgrounds, text, and borders
  3. Update CSS variables if needed
  4. Test color contrast for accessibility (WCAG AA/AAA)

ğŸ“± Responsive Design

Mobile-First Approach

MyDzine is fully responsive and mobile-optimized:

  • Breakpoints: Uses Chakra UI's default breakpoints:
    • base: 0px (mobile)
    • sm: 480px
    • md: 768px (tablet)
    • lg: 992px (desktop)
    • xl: 1280px (large desktop)
  • Responsive Props: All components use Chakra UI responsive props: 
    <Text fontSize={{ base: "lg", md: "xl", lg: "2xl" }} />
<Box padding={{ base: 4, md: 6, lg: 8 }} />

Mobile Navigation

  • Hamburger Menu: Mobile drawer navigation in src/components/layout/Header.tsx
  • Bottom Navigation: Mobile bottom nav bar in src/components/dashboard/DashboardShell.tsx
  • Responsive Grids: Use SimpleGrid with responsive columns: 
    <SimpleGrid columns={{ base: 1, sm: 2, md: 3, lg: 4 }} />

Mobile Optimization Tips

  • âœ… Use clamp() for fluid typography
  • âœ… Optimize images for mobile (Next.js Image component)
  • âœ… Test on iPhone SE (375px width) - smallest common mobile
  • âœ… Ensure touch targets are at least 44x44px
  • âœ… Use safe area insets for devices with notches
