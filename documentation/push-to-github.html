Push Code to GitHub

üì¶ GitHub Setup: This guide will walk you through setting up a GitHub repository and pushing your MyDzine code to GitHub. This is a prerequisite step before deploying to Vercel or AWS EC2.

Requirements

  • GitHub account (free)
  • Git installed on your computer
  • Code editor (VS Code recommended)
  • MyDzine source code

Step 1: Install Git

  1. Download Git from git-scm.com
  2. Install Git following the installation wizard
  3. Configure Git (first time only): 
    git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Step 2: Create GitHub Repository

  1. Go to github.com and sign in
  2. Click "+" ‚Üí "New repository"
  3. Repository name: mydzine (or your preferred name)
  4. Set visibility: Private (recommended) or Public
  5. Do NOT initialize with README (we'll push existing code)
  6. Click "Create repository"
  7. Copy the repository URL (e.g., https://github.com/yourusername/mydzine.git)

Step 3: Initialize Git and Push to GitHub (Command Line)

# Navigate to your MyDzine project folder
cd /path/to/photoshot-main

# Initialize Git repository
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: MyDzine v2.0"

# Rename branch to main (if needed)
git branch -M main

# Add remote repository
git remote add origin https://github.com/yourusername/mydzine.git

# Push to GitHub
git push -u origin main

  • .DS_Store - macOS system files

Step 4: Alternative - GitHub Desktop (Optional)

For users who prefer a GUI interface:

  1. Download and install GitHub Desktop
  2. Sign in to your GitHub account
  3. Click "File" ‚Üí "Add Local Repository"
  4. Select your MyDzine project folder
  5. Click "Publish repository"
  6. Name: mydzine, keep it private, click "Publish Repository"
  7. Commit message: "Initial commit: MyDzine v2.0"
  8. Click "Commit to main" then "Push origin"

Step 5: Verify Upload

  1. Go to your GitHub repository page
  2. Verify all files are uploaded:
    • ‚úÖ package.json
    • ‚úÖ src/ directory
    • ‚úÖ public/ directory
    • ‚úÖ .gitignore
    • ‚úÖ README.md
  3. Verify sensitive files are NOT uploaded:
    • ‚ùå .env.local (should be in .gitignore)
    • ‚ùå node_modules/ (should be in .gitignore)
